## 🔍 **PROPRIÉTÉS DU MODÈLE UTILISÉES**

### **Table: `sales_documents`**
| Onglet | Champs principaux | Validations |
|--------|------------------|-------------|
| **Source** | `parentDocumentId`, `sourceDocumentIds[]` | BL non facturé |
| **Client** | `partnerClientId`, `clientName`, `taxRegime` | ANR valide, crédit OK |
| **Articles** | `lines[]`, `totals`, TVA/retenue | Calculs automatiques |
| **Paiement** | `paymentSchedule[]`, `paymentTermsDays` | Échéancier multi |
| **Validation** | `status`, `exportedToAccounting` | Workflow complet |

## 📌 **FONCTIONNALITÉS CLÉS**

### Liste factures (ECR-FAC-001)
1. **Vue consolidée** : Tous types (factures, avoirs, proforma)
2. **Indicateurs paiement** : Statuts visuels clairs
3. **Actions groupées** : Envoi, relance, lettrage
4. **Détail rapide** : Panneau latéral au clic
5. **Filtres avancés** : Multi-critères

### Création facture (ECR-FAC-002)
1. **Sources multiples** : BL unique/multiple, direct
2. **Types variés** : Standard, acompte, situation, transport
3. **Calculs automatiques** : TVA 19.25%, retenue 1.1%
4. **Échéancier flexible** : Paiement unique ou échelonné
5. **Envoi intégré** : Email avec PDF automatique
6. **Contrôles** : ANR, crédit, validations

## 🎯 **RÈGLES MÉTIER APPLIQUÉES**

- **RG-V050** : Facture depuis BL validé uniquement
- **RG-V051** : TVA 19.25% obligatoire (sauf exonération)
- **RG-V052** : Retenue source 1.1% si applicable
- **RG-V053** : ANR obligatoire pour entreprises
- **RG-V054** : Limite espèces 100K XAF
- **RG-V055** : Échéancier selon conditions client
